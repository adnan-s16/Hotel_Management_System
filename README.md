# Simple Hotel Management System

A lightweight, object-oriented Python application designed to process, map, and organize hotel datasets alongside user booking profiles. This system demonstrates data filtering, custom mapping structures, and runtime sorting capabilities using standard object-oriented principles.

## 🚀 Features

* **Object-Oriented Architecture:** Dedicated, lightweight models managing internal domains for `Hotel` assets and `User` demographic details.
* **Dynamic Dataset Sorting:** Flexible filtering mechanics parsing records instantly by `Name`, `Rating`, or `Room Availability`.
* **Location-Based Queries:** Instant query matching to segment data collections down to target municipalities or cities.
* **Streamlined Data Mapping:** Utilizes standard Python data collection methodologies to tie specific consumer metrics safely to active hotel inventories.

## 📂 Codebase Breakdown

* `Hotel`: Representation layer handling metadata variables (Pricing tiers, Room inventory levels, Geographic tags, Review systems).
* `User`: Transactional layer archiving system configurations (Identity references, Total spending calculations, Profile tags).
* `Sorting Utilities`: Modular processes allowing lists to shift sort contexts implicitly using native Python lambda sorting keys.
